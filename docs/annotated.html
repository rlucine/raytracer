Data Structures
Here are the data structures with brief descriptions:
 CCOLLISIONData about the collision with a shape
 CELLIPSOIDStruct for storing ellipsoid data
 CFACEStruct for storing face data
 CIMAGEStruct for uncompressed RGB image data
 CLIGHTStruct containing all light information. LIGHT_POINT need not define direction. LIGHT_DIRECTED need not define where. Only LIGHT_SPOT requires angle
 CLINEStruct for an infinite line emanating from the origin in the given direction. Direction need not be normalized
 CMATERIALStruct for storing object material data
 CMESHStruct for storing triangle mesh data. One SCENE should have just one mesh which defines every face, but each FACE may reside in its own SHAPE
 CPLANEStruct for a flat plane. The point origin is on the plane and the vectors u and v form the basis of the plane space. u and v need not be normalized
 CRGBStruct for 24-bit RGB colors
 CSCENEStruct for storing all scene data
 CSHAPEGeneralized storage of shape data. The SHAPE_TYPE must correspond exactly to the target of the data pointer. SHAPE_SPHERE corresponds to a SPHERE. SHAPE_ELLIPSOID corresponds to an ELLIPSOID. No further shape types are supported
 CSPHEREStruct for storing sphere data
 CVERTEXStruct for indexing into a MESH
